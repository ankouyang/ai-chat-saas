# 登录流程时序图

本文档基于当前项目的 `NextAuth + CredentialsProvider` 实现，对登录请求从前端表单到生成 session 的流程做顺序说明。

## 关键文件

- `components/sign-in-form.tsx`
- `app/api/auth/[...nextauth]/route.ts`
- `lib/auth.ts`

## 时序图

```text
用户
  |
  | 1. 打开 /sign-in
  v
sign-in 页面
  |
  | 2. 输入 email 和 name
  | 3. 调用 signIn("credentials", { email, name, callbackUrl: "/chat", redirect: false })
  v
next-auth/react 的 signIn()
  |
  | 4. NextAuth 客户端内部发起认证请求
  v
/api/auth/callback/credentials
  |
  | 5. 请求命中 app/api/auth/[...nextauth]/route.ts
  | 6. route.ts 把请求交给 NextAuth(authOptions)
  v
NextAuth
  |
  | 7. 识别 provider = credentials
  | 8. 调用 lib/auth.ts 中 CredentialsProvider 的 authorize(credentials)
  v
authorize(credentials)
  |
  | 9. 检查 DATABASE_URL 和 AUTH_SECRET
  | 10. 清洗输入
  |     - email.trim().toLowerCase()
  |     - name.trim()
  | 11. email 为空则返回 null
  | 12. 使用 prisma.user.upsert(...) 查找或创建用户
  v
Prisma / Database
  |
  | 13. 返回 user { id, email, name }
  v
NextAuth
  |
  | 14. 执行 jwt callback
  |     - token.id = user.id
  | 15. 生成 session/jwt
  | 16. 写入登录态相关 cookie
  v
浏览器
  |
  | 17. 前端拿到 signIn 返回结果 result
  | 18. 如果成功，跳转到 callbackUrl = /chat
  v
后续页面或服务端
  |
  | 19. 读取 session，比如 useSession() 或 auth()
  v
/api/auth/session
  |
  | 20. NextAuth 读取 token
  | 21. 执行 session callback
  |     - session.user.id = token.id
  v
最终 session
  |
  | 22. 得到 session.user.id / email / name
  v
完成登录
```

## 每一层分别负责什么

### 1. 前端表单层

`components/sign-in-form.tsx` 负责收集用户输入，并调用：

```ts
signIn("credentials", {
  email,
  name,
  callbackUrl: "/chat",
  redirect: false,
});
```

这里没有手写 `fetch("/api/auth/...")`，因为请求由 NextAuth 客户端内部封装。

### 2. 路由入口层

`app/api/auth/[...nextauth]/route.ts` 负责接住所有 `/api/auth/*` 请求：

```ts
const handler = NextAuth(authOptions);

export { handler as GET, handler as POST };
```

这个文件不直接写登录逻辑，它只是把请求交给 NextAuth。

### 3. 认证规则层

`lib/auth.ts` 里定义了本项目自己的认证规则：

- 使用 `CredentialsProvider`
- 在 `authorize(credentials)` 中校验和清洗输入
- 用 Prisma 查询或创建用户
- 在 `jwt` callback 里把 `user.id` 写到 token
- 在 `session` callback 里把 `token.id` 写到 `session.user.id`

## 当前项目里最关键的点

- `route.ts` 只是统一入口，不是认证核心
- 真正判断“这个用户能不能登录”的地方是 `lib/auth.ts` 里的 `authorize()`
- `/api/auth/callback/credentials` 这条接口的主体流程由 NextAuth 内部实现
- 你只需要提供 provider 配置和回调函数

## 一句话总结

当前项目的登录流程是：

`sign-in 表单 -> signIn("credentials") -> /api/auth/callback/credentials -> route.ts -> NextAuth -> authorize() -> Prisma -> jwt/session -> /chat`
