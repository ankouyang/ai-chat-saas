# 聊天读写流程时序图

本文档基于当前项目已经切换到 `ChatWorkspace + /api/chat` 的实现，对聊天页如何读取 `Chat` / `Message`，以及用户发送消息后如何实时显示和最终落库做顺序说明。

## 关键文件

- `app/chat/page.tsx`
- `components/chat-workspace.tsx`
- `components/app-sidebar.tsx`
- `app/api/chat/route.ts`
- `lib/chat.ts`
- `lib/ai.ts`
- `lib/db.ts`

## 场景一：打开聊天页并读取会话

```text
用户
  |
  | 1. 打开 /chat 或 /chat?chatId=xxx
  v
app/chat/page.tsx
  |
  | 2. 调用 auth() 获取当前 session
  | 3. 未登录则 redirect("/sign-in")
  | 4. 读取 searchParams.chatId
  v
Prisma / Database
  |
  | 5. 查询当前用户的所有 Chat
  |    prisma.chat.findMany({
  |      where: { userId: session.user.id },
  |      orderBy: { updatedAt: "desc" }
  |    })
  | 6. 如果 URL 中有 chatId，则查询当前会话
  |    prisma.chat.findFirst({
  |      where: { id: chatId, userId: session.user.id },
  |      select: { id, title, messages }
  |    })
  v
app/chat/page.tsx
  |
  | 7. 把 chats 传给 AppSidebar
  | 8. 把 activeChat.messages 作为 initialMessages 传给 ChatWorkspace
  | 9. 把 activeChat.id 作为 initialChatId 传给 ChatWorkspace
  v
聊天页完成首屏渲染
```

## 场景二：点击“新建对话”

```text
用户
  |
  | 1. 点击侧边栏“新建对话”
  v
components/app-sidebar.tsx
  |
  | 2. form action={createChatAction}
  v
app/chat/actions.ts -> createChatAction()
  |
  | 3. 调用 auth() 校验当前用户
  | 4. 未登录则 redirect("/sign-in")
  | 5. 执行 prisma.chat.create({
  |      data: {
  |        title: "新对话",
  |        userId: session.user.id
  |      }
  |    })
  v
Prisma / Database
  |
  | 6. 写入一条新的 Chat 记录
  | 7. 返回新 chat.id
  v
createChatAction()
  |
  | 8. revalidatePath("/chat")
  | 9. redirect(`/chat?chatId=${chat.id}`)
  v
聊天页重新加载并显示空会话
```

## 场景三：发送消息并按 chunk 实时显示

```text
用户
  |
  | 1. 在聊天输入框输入内容并点击“发送”
  v
components/chat-workspace.tsx
  |
  | 2. handleSubmit(event)
  | 3. 先做本地 optimistic update：
  |    - 立即插入一条 user 消息
  |    - 立即插入一条空 assistant 占位消息
  | 4. fetch("/api/chat", {
  |      method: "POST",
  |      body: { content, chatId }
  |    })
  v
app/api/chat/route.ts
  |
  | 5. auth() 校验当前用户
  | 6. 调用 ensureChatForUser()：
  |    - 复用当前 chatId
  |    - 或为当前用户新建一条 Chat
  | 7. 先把 user Message 写入数据库
  | 8. 立即发送 meta 事件：event: meta / data: { chatId }
  | 9. 调用 generateChatReplyStream(chatId, onChunk)
  v
lib/ai.ts
  |
  | 10. 根据 chatId 查询最近历史消息
  | 11. 组装 system + user/assistant 上下文
  | 12. 向模型发起 stream: true 请求
  | 13. 每收到一个 delta，就通过 onChunk(chunk) 往上抛
  v
app/api/chat/route.ts
  |
  | 14. onChunk(chunk) -> writeEvent("delta", { content: chunk })
  | 15. 不等待完整结果，持续把 chunk 推给前端
  v
components/chat-workspace.tsx
  |
  | 16. response.body.getReader() 持续读取 SSE
  | 17. 收到 meta:
  |     - setChatId(chatId)
  |     - router.replace(`/chat?chatId=${chatId}`)
  | 18. 收到 delta:
  |     - appendAssistantChunk(chunk)
  |     - 当前 assistant 占位消息立刻追加这段文本
  | 19. 流式过程中不使用新的 initialMessages 覆盖本地消息
  |     - 避免 UI 看起来像最后一次性出现
  v
用户
  |
  | 20. 看到 assistant 内容按 chunk 持续增长
  v
app/api/chat/route.ts
  |
  | 21. 模型结束后，把完整 assistant Message 写入数据库
  | 22. 更新 Chat.title
  | 23. writeEvent("done", { chatId, content: assistantContent })
  | 24. revalidatePath("/chat")
  v
components/chat-workspace.tsx
  |
  | 25. 收到 done，补齐最终完整文本
  | 26. router.refresh()
  v
app/chat/page.tsx
  |
  | 27. 重新查库，让页面状态与数据库最终一致
  v
聊天页完成一次“实时渲染 + 最终落库同步”
```

## 数据关系在这个流程里的体现

### 1. User -> Chat

创建会话时：

```ts
await prisma.chat.create({
  data: {
    title: "新对话",
    userId: session.user.id,
  },
});
```

这里的 `userId` 表示：这条 `Chat` 属于当前登录用户。

### 2. Chat -> Message

创建消息时：

```ts
await prisma.message.create({
  data: {
    role: "user",
    content,
    chatId,
  },
});
```

这里的 `chatId` 表示：这条 `Message` 挂在当前会话下面。

## 当前实现的几个关键点

- 聊天页读取会话时，查询条件里同时包含 `chatId` 和 `userId`
- 这意味着用户只能读到自己的会话，不能通过改 URL 读取别人的数据
- 第一次发消息时，如果还没有会话，系统会自动先创建 `Chat`
- assistant 的主显示路径是前端 chunk 追加，不是等数据库回刷后才显示
- `router.refresh()` 只负责最终一致性同步，不负责主要展示

## 一句话总结

当前项目的聊天读写流程是：

`聊天页读取 session -> 查询当前用户的 chats 和 activeChat -> ChatWorkspace 发起 /api/chat -> 路由先写 user Message -> 模型 chunk 持续推给前端 -> 前端按 chunk 追加 assistant 文本 -> 结束后 assistant 消息落库并 refresh 同步`
