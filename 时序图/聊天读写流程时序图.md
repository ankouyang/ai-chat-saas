# 聊天读写流程时序图

本文档基于当前项目的 Day 2 后半段实现，对聊天页如何读取 `Chat` / `Message`，以及用户发送消息后如何写入数据库做顺序说明。

## 关键文件

- `app/chat/page.tsx`
- `app/chat/actions.ts`
- `components/app-sidebar.tsx`
- `components/chat-input.tsx`
- `lib/db.ts`

## 场景一：打开聊天页并读取会话

```text
用户
  |
  | 1. 打开 /chat 或 /chat?chatId=xxx
  v
app/chat/page.tsx
  |
  | 2. 调用 auth() 获取当前 session
  | 3. 未登录则 redirect("/sign-in")
  | 4. 读取 searchParams.chatId
  v
Prisma / Database
  |
  | 5. 查询当前用户的所有 Chat
  |    prisma.chat.findMany({
  |      where: { userId: session.user.id },
  |      orderBy: { updatedAt: "desc" }
  |    })
  | 6. 如果 URL 中有 chatId，则查询当前会话
  |    prisma.chat.findFirst({
  |      where: { id: chatId, userId: session.user.id },
  |      select: { id, title, messages }
  |    })
  v
app/chat/page.tsx
  |
  | 7. 把 chats 传给侧边栏
  | 8. 把 activeChat.messages 渲染到消息列表
  | 9. 把 activeChat.id 传给聊天输入框
  v
聊天页完成首屏渲染
```

## 场景二：点击“新建对话”

```text
用户
  |
  | 1. 点击侧边栏“新建对话”
  v
components/app-sidebar.tsx
  |
  | 2. form action={createChatAction}
  v
app/chat/actions.ts -> createChatAction()
  |
  | 3. 调用 auth() 校验当前用户
  | 4. 未登录则 redirect("/sign-in")
  | 5. 执行 prisma.chat.create({
  |      data: {
  |        title: "新对话",
  |        userId: session.user.id
  |      }
  |    })
  v
Prisma / Database
  |
  | 6. 写入一条新的 Chat 记录
  | 7. 返回新 chat.id
  v
createChatAction()
  |
  | 8. revalidatePath("/chat")
  | 9. redirect(`/chat?chatId=${chat.id}`)
  v
聊天页重新加载并显示空会话
```

## 场景三：发送第一条消息

```text
用户
  |
  | 1. 在输入框中输入内容并点击“发送”
  v
components/chat-input.tsx
  |
  | 2. form action={sendMessageAction}
  | 3. 提交字段：
  |    - content
  |    - chatId（如果当前没有会话，这里可能为空）
  v
app/chat/actions.ts -> sendMessageAction(formData)
  |
  | 4. 调用 auth() 校验当前用户
  | 5. 读取 content 和 chatId
  | 6. content 为空则直接 redirect 回聊天页
  |
  | 7. 如果 chatId 为空：
  |    - 先创建一条 Chat
  |    - title 取用户消息前 24 个字符
  |    - userId = session.user.id
  v
Prisma / Database
  |
  | 8. 写入 Chat（仅第一次消息时）
  | 9. 写入 user Message
  | 10. 写入 assistant Message（当前为演示回复）
  | 11. 更新 Chat.title
  v
sendMessageAction()
  |
  | 12. revalidatePath("/chat")
  | 13. redirect(`/chat?chatId=${chat.id}`)
  v
app/chat/page.tsx
  |
  | 14. 重新查询当前 Chat 和 messages
  v
页面显示最新消息
```

## 数据关系在这个流程里的体现

### 1. User -> Chat

创建会话时：

```ts
await prisma.chat.create({
  data: {
    title: "新对话",
    userId: session.user.id,
  },
});
```

这里的 `userId` 表示：这条 `Chat` 属于当前登录用户。

### 2. Chat -> Message

创建消息时：

```ts
await prisma.message.create({
  data: {
    role: "user",
    content,
    chatId,
  },
});
```

这里的 `chatId` 表示：这条 `Message` 挂在当前会话下面。

## 当前实现的几个关键点

- 聊天页读取会话时，查询条件里同时包含 `chatId` 和 `userId`
- 这意味着用户只能读到自己的会话，不能通过随便改 URL 读取别人的数据
- 第一次发消息时，如果还没有会话，系统会自动先创建 `Chat`
- 当前 assistant 回复是服务端拼出来的演示文本，还没有接真实模型

## 一句话总结

当前项目的聊天读写流程是：

`聊天页读取 session -> 查询当前用户的 chats -> 选择 activeChat -> 表单提交 sendMessageAction -> 先确保 Chat 存在 -> 写入 Message -> redirect 回 /chat?chatId=xxx -> 页面重新查库并显示结果`
